## 1. Общее описание

Telegram-бот для бегового клуба айтишников школы 21. Участники ставят себе беговые челленджи, отчитываются о тренировках, участвуют в мероприятиях и турнирах. Система геймификации (XP, уровни, стрики) мотивирует бегать регулярно.

Стек: Python 3.12, aiogram 3.10, SQLAlchemy 2.0 (async), SQLite (aiosqlite), APScheduler.

Бот работает в двух режимах:
- Личные сообщения (регистрация, создание челленджей, профиль)
- Групповой чат с топиками (отчёты, анонсы мероприятий, дайджест)

## 3. Модуль 1: Регистрация и профиль

### Регистрация (FSM)

При первом /start пользователь проходит регистрацию:

1. Бот спрашивает: "Ты уже когда-то регистрировался?"
   - "Да" → бот сообщает что профиль не найден, предлагает создать новый
   - "Нет" → переходит к вводу ника
2. Ввод школьного ника (school_nick):
   - Только латинские буквы (a-z), без цифр, дефисов, символов
   - От 3 до 20 символов
   - Уникальный в системе
   - Приводится к lowercase
3. Ввод имени и фамилии (full_name):
   - Свободный текст
   - Можно пропустить (ввести "-")
4. Профиль создан, показывается главное меню

При повторном /start бот узнаёт пользователя и показывает главное меню.

### Профиль

Кнопка "👤 Мой профиль" показывает:

```

Активные челленджи:
  📜 Вернуться к бегу
```

### Главное меню (кнопки)

## 4. Модуль 2: Челленджи

### 5 типов челленджей

#### 1. Регулярный (weekly_runs)
- Цель: N пробежек в неделю
- Условие зачёта одной пробежки (ИЛИ-логика):
  - Минимум X километров за пробежку (опционально)
  - Минимум Y минут активности за пробежку (опционально)
  - Если заданы оба — пробежка засчитывается при выполнении ЛЮБОГО из них
  - Хотя бы одно условие обязательно
- Дата старта (обязательно, "-" = сегодня, не может быть в прошлом)
- Дата окончания (опционально, "-" = бессрочный)
- Пример: "3 пробежки в неделю, минимум 3 км или 20 минут каждая, с 01.06.2026 бессрочно"

#### 2. Дневной спринт (daily_km)
- Цель: набегать N км за один день
- Дата старта (обязательно)
- Дата окончания: автоматически = старт + 1 день
- Пример: "Набегать 10 км за день 15.06.2026"

#### 3. Недельный спринт (weekly_km)
- Цель: набегать N км за неделю
- Дата старта (обязательно)
- Дата окончания: автоматически = старт + 7 дней
- Пример: "Набегать 50 км за неделю начиная с 01.07.2026"

#### 4. Месячный спринт (monthly_km)
- Цель: набегать N км за месяц
- Дата старта (обязательно)
- Дата окончания: автоматически = старт + 30 дней
- Пример: "Набегать 200 км за месяц"

#### 5. Забег (race)
- Цель: пробежать конкретный забег
- Дистанция в км (обязательно)
- Целевое время в минутах (опционально, "-" = без ограничения)
- Дата забега (обязательно, не может быть в прошлом)
- Пример: "Полумарафон 21.1 км за 120 минут, 15.09.2026"

### Общие параметры всех челленджей

- Название (3-80 символов)
- Цена слова / штраф (опционально): текст, например "Куплю всем пиццу"
- Публичность: все челленджи по умолчанию публичные
- Любой участник может создать челлендж

### Присоединение к чужим челленджам

- Команда /public_challenges показывает список публичных челленджей
- Каждый челлендж имеет кнопку "🤝 Присоединиться"
- У присоединившегося свой отдельный прогресс
- Нельзя присоединиться к своему челленджу

### Пауза челленджа (форс-мажор)

- Только администратор может поставить челлендж на паузу
- Указывается количество дней паузы
- Во время паузы пробежки не засчитываются в этот челлендж
- После паузы челлендж продолжается

### Закрытие челленджа

- Спринты (daily/weekly/monthly): автоматически при достижении цели
- Регулярный: по дате окончания или бессрочный (можно закрыть вручную)
- Забег: когда подан и одобрен отчёт с результатом

### FSM создания челленджа (порядок вопросов)

```
[Все типы]     Название
[Все типы]     Выбор типа
                ↓
[weekly_runs]  Мин. км за пробежку (или "-")
[weekly_runs]  Мин. минут за пробежку (или "-")
[weekly_runs]  Пробежек в неделю
[weekly_runs]  Дата старта
[weekly_runs]  Дата окончания (или "-" = бессрочный)
[weekly_runs]  Цена слова
                ↓
[спринты]      Цель в км
[спринты]      Дата старта
[спринты]      Цена слова (deadline считается автоматически)
                ↓
[race]         Дистанция в км
[race]         Целевое время в минутах (или "-")
[race]         Дата забега
[race]         Цена слова
```

## 5. Модуль 3: Отчёты и P2P проверка

### Подача отчёта

Отчёты принимаются ТОЛЬКО в топике REPORTS_THREAD_ID основной группы.

Формат: фото (скриншот трекера) с подписью:
```
#отчет 7.5
```
где 7.5 — дистанция в км. Если число не указано, засчитывается 1 км.

### Привязка к челленджу

После подачи отчёта бот анализирует активные челленджи пользователя и предлагает привязать отчёт к подходящему челленджу через инлайн-кнопки. Пользователь может:
- Выбрать челлендж из списка
- Не привязывать (просто тренировка для XP и таблицы)

### P2P голосование

После подачи отчёта бот отвечает сообщением с кнопками:
```
🏃 Новый отчёт!
Атлет: quicksir
Результат: 7.5 км

Нужно 3 голоса или одобрение админа.

[✅ Засчитать] [❌ Фейк]
[👑 Админ: одобрить]
```

Правила голосования:
- Нельзя голосовать за свой отчёт
- Один голос с человека (UniqueConstraint в БД)
- 3 голоса (VOTES_REQUIRED из config.py) = автоматическое одобрение
- Админ или модератор может одобрить мгновенно (1 клик)
- Админ или модератор может отклонить отчёт

### После одобрения отчёта

1. Начисляется XP: km × XP_PER_KM (по умолчанию 10 XP за км)
2. Если отчёт привязан к мероприятию: XP × event.xp_multiplier + event.xp_bonus
3. Обновляется прогресс всех подходящих активных челленджей (services/challenges.py update_on_report)
4. Проверяется личный рекорд (services/xp.py check_and_update_pr)
5. Если новый PR — бонус XP_PR_BONUS и уведомление
6. Обновляется level пользователя (xp // 100)
7. Бот отправляет уведомление в личку

### Типы отчётов

- "training" — обычная тренировка
- "event" — участие в мероприятии (event_id обязателен, применяется XP-множитель)

## 6. Модуль 4: Мероприятия

### Шаблоны мероприятий

Шаблоны хранятся в БД, управляются через админ-меню. Параметры:
- Название (например: "5 вёрст", "Long Run")
- Описание
- Правила участия (текст)
- Информация о регистрации
- Тип: "мы организаторы" (is_external=False) или "мы гости" (is_external=True)
- XP бонус за участие (по умолчанию 100)
- XP множитель за км (по умолчанию 1.5)

### Создание мероприятия

Доступно любому участнику. FSM:
1. Название
2. Описание (или "-")
3. Место (или "-")
4. Дата и время (ДД.ММ.ГГГГ ЧЧ:ММ)
5. Дистанция в км (или "-")

### Анонс мероприятия

После создания бот публикует анонс в двух местах:

**Основная группа** (EVENTS_THREAD_ID) — с кнопками регистрации:
```
📣 АНОНС: Long Run 10км
⏰ 22.06.2026 09:00
📍 Парк Чернобыльца
🏁 10 км

Описание мероприятия...

[🏃 Участвую (5)] [❌ Не пойду (2)]
```

**Школьная группа** (SECONDARY_GROUP_ID) — текстовый анонс БЕЗ кнопок:
```
📣 АНОНС: Long Run 10км
⏰ 22.06.2026 09:00
📍 Парк Чернобыльца
🏁 10 км

Хочешь бегать вместе? 👇
Вступай в наш беговой клуб: https://t.me/+HLikNXKlA3YwNDRi
```

### Регистрация на мероприятие

Кнопки со счётчиками:
- "🏃 Участвую (N)" — статус "going"
- "❌ Не пойду (N)" — статус "not_going"

Счётчики обновляются в реальном времени при каждом нажатии. Можно менять решение (going ↔ not_going).

### Отчёт по мероприятию

После мероприятия участник подаёт отчёт как обычно (#отчет), но привязывает его к event_id. При одобрении применяется повышенный XP (множитель + бонус из шаблона или события).

## 7. Модуль 5: Турниры

### Создание турнира (только админ)

Параметры:
- Название
- Тип: "km" (кто больше км) / "minutes" (кто больше минут) / "days" (кто больше дней подряд) / "team_km" (командный)
- Длительность (по умолчанию 7 дней)

### Участие

- Бот публикует анонс турнира с кнопкой "🏆 Принять вызов"
- Участники регистрируются нажатием кнопки
- Очки обновляются автоматически при одобрении отчётов

### Финализация

По окончании срока турнира (автоматически через scheduler):
1. Определяется победитель по score
2. Распределяются XP за призовые места:
   - 1 место: 250 XP
   - 2 место: 150 XP
   - 3 место: 100 XP
3. Публикуется таблица результатов в группу

## 8. Модуль 6: Таблица лидеров и система уровней

### Система уровней (all-time)

```
Level = total_xp // 100
```

Уровень НИКОГДА не сбрасывается. Это постоянное достижение.

Категории:
- 🌱 Новичок (Level 0-4, 0-499 XP)
- 🌿 Бегун (Level 5-19, 500-1999 XP)
- 🌳 Атлет (Level 20+, 2000+ XP)

### Сезонный XP (квартальный)

Отдельное поле season_xp, сбрасывается каждый квартал (01.01, 01.04, 01.07, 01.10). Позволяет новичкам конкурировать со старожилами в рамках сезона.

### Таблицы лидеров

**All-time таблица** (по уровню):
```
🏆 ТОП БЕГОВОЙ КЛУБ

🥇 quicksir (@vasya)
   Level 47 • 4750 XP 🔥15
🥈 leeannaf (@leeanna)
   Level 23 • 2350 XP 🔥8
```

**Сезонная таблица** (по season_xp):
```
📊 СЕЗОН Q3 2026

🥇 newrunner (@alex)
   450 Season XP
🥈 quicksir (@vasya)
   380 Season XP
```

### Источники XP

| Событие | XP |
|---|---|
| 1 км тренировки | 10 |
| Закрытая неделя (регулярный челлендж) | 50 |
| Участие в мероприятии клуба (бонус) | 100 |
| Участие в мероприятии-гость (бонус) | 75 |
| Личный рекорд | 50 |
| Milestone стрик 4 недели | 100 |
| Milestone стрик 8 недель | 150 |
| Milestone стрик 12 недель | 200 |
| Milestone стрик 20 недель | 300 |
| Победа в турнире (1 место) | 250 |
| 2 место в турнире | 150 |
| 3 место в турнире | 100 |

### Стрики

Стрик — количество недель подряд, когда пользователь выполнил норму регулярного челленджа. Считается в еженедельном дайджесте (воскресенье 21:00). Если норма не выполнена — стрик сбрасывается в 0.

## 9. Модуль 7: Еженедельный дайджест

Публикуется автоматически каждое воскресенье в 21:00 МСК в топик DIGEST_THREAD_ID.

### Содержание дайджеста

```
📊 ДАЙДЖЕСТ НЕДЕЛИ — IT БЕГОТНЯ 21
Период: 09.06 – 15.06.2026

🏃 АКТИВНОСТЬ КЛУБА
Всего км: 187.3  |  Тренировок: 34  |  Участников: 12

✅ ГЕРОИ НЕДЕЛИ (выполнили норму)
• quicksir — 5 тренировок (27.3 км) 🔥8
• leeannaf — 3 тренировки (15.1 км) 🔥3

🍕 ДОЛЖНИКИ (не выполнили норму)
• slowrunner (1/3) — 💰 пицца на команду

🏆 ТОП-5 ПО XP
🥇 quicksir — 4750 XP 🔥8
🥈 leeannaf — 2350 XP 🔥3
...

🔥 MILESTONE СТРИКИ
quicksir — 8 недель без пропусков! (+150 XP)

📅 МЕРОПРИЯТИЯ НА ЭТОЙ НЕДЕЛЕ
✅ 5 вёрст — 8 участников
✅ Long Run — 5 участников

🏆 ТУРНИР НЕДЕЛИ
🥇 runner1 — 52 км (+250 XP)
...

Новая неделя — новые километры! Вперёд 🏃💪
```

### Действия при публикации дайджеста

1. Подсчёт статистики за неделю
2. Определение героев и должников по регулярным челленджам
3. Обновление стриков: +1 для героев, сброс для должников
4. Начисление XP за закрытую неделю (XP_PER_WEEK)
5. Проверка и начисление milestone-бонусов (STREAK_MILESTONES)
6. Сброс недельных счётчиков (current_runs, current_value для weekly_runs)
7. Публикация текста в группу

## 10. Модуль 8: Администрирование

### Доступ

- Администраторы: определяются через ADMIN_IDS в .env
- Модераторы: хранятся в таблице Moderator, добавляются администратором
- Модератор может: одобрять/отклонять отчёты
- Администратор может: всё что модератор + управлять модераторами, шаблонами, паузить челленджи

### Команды

- /admin — открыть админ-панель (только для админов)

### Админ-панель (кнопки)

```
[📅 Создать мероприятие]
[👥 Управление модераторами]
[📋 Шаблоны мероприятий]
[🏆 Создать турнир]
[⬅️ Главное меню]
```

### Управление модераторами

- Добавить модератора (по Telegram ID)
- Удалить модератора
- Список модераторов

### Пауза челленджа (форс-мажор)

Админ может поставить все челленджи пользователя на паузу на N дней (например: болезнь, травма). Во время паузы пробежки не засчитываются, стрик не сбрасывается.

## 11. Конфигурация (.env)

```env
# Telegram Bot
BOT_TOKEN=your_bot_token_here
PROXY_URL=socks5://login:password@host:port

# Database
DB_URL=sqlite+aiosqlite:///./it_run.db

# Main group (внутренний клуб)
GROUP_ID=-100123456789
REPORTS_THREAD_ID=123
EVENTS_THREAD_ID=456
DIGEST_THREAD_ID=789

# Secondary group (школьный чат для репостов анонсов)
SECONDARY_GROUP_ID=-100987654321
SECONDARY_THREAD_ID=

# Admins (Telegram ID через запятую)
ADMIN_IDS=123456789,987654321

# Game settings
VOTES_REQUIRED=3
XP_PER_KM=10
XP_PER_WEEK=50
XP_PR_BONUS=50
```

Все значения по умолчанию:

## 13. Статус реализации

| Модуль | Статус | Что готово | Что осталось |
|---|---|---|---|
| Регистрация | ✅ Готово | FSM, профиль, таблица лидеров | — |
| Челленджи | 🔧 80% | 5 типов, сервис, FSM, присоединение | Правка логики дат |
| Отчёты | ❌ Не начато | — | services + handlers |
| Мероприятия | ❌ Не начато | — | services + handlers + шаблоны + анонсы |
| Турниры | ❌ Не начато | — | services + handlers + финализация |
| Лидеры | 🔧 50% | All-time таблица | Сезонная таблица |
| Дайджест | ❌ Не начато | — | services + scheduler |
| Администрирование | ❌ Не начато | — | handlers/admin.py |

### План реализации по чатам

1. Чат 1: Правка дат челленджей (keyboards, services/challenges, handlers/challenges)
2. Чат 2: Модуль отчётов (services/reports, handlers/reports)
3. Чат 3: Модуль мероприятий (services/events, handlers/events, handlers/admin шаблоны)
4. Чат 4: Турниры + Дайджест (services/tournaments, services/digest, services/scheduler)
5. Чат 5: Админ-панель + финальная сборка + middleware авторегистрации
