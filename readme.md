# IT БЕГОТНЯ 21 — Telegram-бот бегового клуба

Бот для внутреннего бегового клуба айтишников Школы 21. Участники ставят беговые челленджи, отчитываются о тренировках, участвуют в мероприятиях и турнирах. Геймификация (XP, уровни, стрики) мотивирует бегать регулярно.

**Стек:** Python 3.12 · aiogram 3.10 · SQLAlchemy 2.0 async · SQLite (aiosqlite) · APScheduler · Docker

## Быстрый старт

### Требования

- Docker + Docker Compose

### Запуск

```bash
git clone <repo>
cd runclubbot

# Создай .env (см. раздел Конфигурация)
cp .env.example .env
nano .env

# Создай папку для БД
mkdir -p data

# Сборка и запуск
docker compose build
docker compose up -d

# Логи
docker compose logs -f
```

### Обновление

```bash
# Загрузи новые файлы, затем:
docker compose build --no-cache && docker compose up -d
```

### Управление

```bash
docker compose ps          # статус
docker compose logs -f     # логи в реальном времени
docker compose down        # остановка
docker compose restart     # перезапуск
```

### Резервная копия БД

База лежит в `./data/it_run.db` на хосте. Добавь в cron:

```bash
0 3 * * * cp /path/to/runclubbot/data/it_run.db /path/to/backups/it_run_$(date +\%Y\%m\%d).db
```

## Конфигурация

Файл `.env` в корне проекта:

```env
# Telegram Bot
BOT_TOKEN=your_bot_token_here
PROXY_URL=http://host:port          # опционально, если нет прямого доступа к Telegram

# Database (путь внутри контейнера)
DB_URL=sqlite+aiosqlite:////app/data/it_run.db

# Основная группа клуба
GROUP_ID=-100123456789
REPORTS_THREAD_ID=263               # ID топика «Отчёты»
EVENTS_THREAD_ID=49                 # ID топика «Мероприятия»
DIGEST_THREAD_ID=1                  # ID топика «Дайджест» (1 = General)

# Дополнительная группа (Школа 21) — только для репостов анонсов
SECONDARY_GROUP_ID=-100987654321
SECONDARY_THREAD_ID=4

# Администраторы (Telegram ID через запятую)
ADMIN_IDS=123456789,987654321

# Игровые настройки
VOTES_REQUIRED=3                    # голосов для одобрения отчёта
XP_PER_KM=10                       # XP за 1 км
XP_PER_WEEK=50                     # XP за закрытую неделю
XP_PR_BONUS=50                     # XP за личный рекорд
```

**Milestone-стрики** (хардкод в `config.py`):

| Недель подряд | Бонус XP |
|---|---|
| 4  | 100 |
| 8  | 150 |
| 12 | 200 |
| 20 | 300 |

```
runclubbot/

├── .env
│
├── handlers/               # Telegram-логика (FSM, callbacks, команды)
│   ├── __init__.py         # Сборка всех роутеров
│   ├── common.py           # /start, регистрация, профиль, лидеры
│   ├── challenges.py       # Создание и просмотр челленджей
│   ├── reports.py          # Приём отчётов, голосование, апрув/реджект
│   ├── events.py           # Мероприятия, регистрация участников
│   ├── tournaments.py      # Турниры: создание, таблица, регистрация
│   └── admin.py            # Админ-панель
│
└── services/               # Бизнес-логика (не знает о Telegram)
    ├── users.py            # CRUD пользователей, XP, стрики
    ├── xp.py               # Начисление XP, проверка PR
    ├── challenges.py       # Логика челленджей, прогресс, is_run_counts
    ├── reports.py          # Создание отчётов, голосование, апрув
    ├── events.py           # Логика мероприятий и шаблонов
    ├── tournaments.py      # Логика турниров, update_score, финализация
    ├── digest.py           # Еженедельный дайджест + личные отчёты
    └── scheduler.py        # APScheduler: дайджест, таблица, финализация
```

**Принцип разделения:**
- `handlers/` — только Telegram. Принимает сообщение, вызывает сервис, отвечает.
- `services/` — только бизнес-логика и БД. Ничего не знает о Telegram.
- `keyboards.py` — все кнопки в одном файле.
- `database.py` — накатывает `ALTER TABLE` миграции при каждом старте.

## Модули

### Регистрация и профиль

При первом `/start` — FSM-регистрация: школьный ник (латиница, 3–20 символов) + имя. При повторном `/start` — сразу главное меню.

Кнопка **«👤 Мой профиль»** показывает уровень, XP, стрик, PR, активные челленджи.

### Челленджи

5 типов:

| Тип | Описание |
|---|---|
| `weekly_runs` | N пробежек в неделю (с мин. км или мин. минутами) |
| `daily_km` | N км за один день |
| `weekly_km` | N км за неделю |
| `monthly_km` | N км за 30 дней |
| `race` | Конкретный забег на дату |

Создание через FSM в личке бота. Все челленджи публичные — любой участник может присоединиться через `/public_challenges`. У каждого участника свой прогресс.

Администратор может поставить челлендж на паузу (болезнь, травма) на N дней — стрик при этом не сбрасывается.

### Отчёты и голосование

Отчёты принимаются **только в топике REPORTS_THREAD_ID**.

**Формат:**
```
#отчет 7.5

Описание тренировки...
```
Можно с фото (скриншот трекера) или текстом. Дистанция берётся из первой строки.

**Флоу после подачи:**
1. Бот предлагает привязать к мероприятию или челленджам (если есть активные)
2. Сообщение с кнопками голосования публикуется в чате
3. 3 голоса «✅ Засчитать» = автоодобрение; 3 голоса «❌ Фейк» = отклонение
4. Админ/модератор может одобрить или отклонить в один клик
5. При одобрении: начисляется XP, обновляется прогресс челленджей и турнира, публикуется уведомление в General

Нельзя голосовать за свой отчёт. Один голос с человека.

### Мероприятия

Создаются через FSM (доступно всем участникам). После создания бот публикует анонс:
- В топик `EVENTS_THREAD_ID` — с кнопками «🏃 Участвую / ❌ Не пойду» и живыми счётчиками
- В `SECONDARY_GROUP_ID` — текстовый анонс со ссылкой на вступление в клуб

Шаблоны мероприятий (с XP-множителем и бонусом) управляются через админ-панель.

### Турниры

Создаются администратором через FSM (`/tournament_new`). 4 типа:
- `km` — кто больше километров
- `minutes` — кто больше минут
- `days` — кто больше дней с тренировкой
- `team_km` — командные километры

После регистрации (кнопка «🏆 Принять вызов» в анонсе) очки начисляются автоматически при каждом одобренном отчёте.

**Каждый день в 21:00 (Asia/Omsk)** — публикуется текущая таблица лидеров турнира.

По окончании срока — автофинализация каждые 15 минут:
- 1 место: **250 XP**
- 2 место: **150 XP**
- 3 место: **100 XP**

### Система XP и уровней

```
Level = total_xp // 100
```

| Уровень | XP |
|---|---|
| 🌱 Новичок | 0–499 |
| 🌿 Бегун | 500–1999 |
| 🌳 Атлет | 2000+ |

Уровень никогда не сбрасывается. Дополнительно есть **сезонный XP** (сбрасывается раз в квартал) для честной конкуренции новичков со старожилами.

**Источники XP:**

| Событие | XP |
|---|---|
| 1 км тренировки | 10 |
| Закрытая неделя | 50 |
| Личный рекорд | 50 |
| Мероприятие клуба (бонус) | 100 |
| Milestone 4 недели | 100 |
| Milestone 8 недель | 150 |
| Milestone 12 недель | 200 |
| Milestone 20 недель | 300 |
| 1 место в турнире | 250 |
| 2 место в турнире | 150 |
| 3 место в турнире | 100 |

### Еженедельный дайджест

Публикуется **каждое воскресенье в 21:00 Asia/Omsk** в General группы.

Содержит:
- Активность клуба за неделю (км, тренировок, участников)
- Герои недели (выполнили норму) + обновление стриков и начисление XP
- Должники (не выполнили норму) + сброс стрика в 0
- Топ-5 по XP
- Milestone-стрики с бонусным XP
- Мероприятия недели
- Итоги завершённых турниров

После публикации: сброс недельных счётчиков `current_runs/current_value`.

Каждому участнику с активными челленджами отправляется **личный отчёт** о прогрессе в ЛС.

## База данных

SQLite, файл `./data/it_run.db` (монтируется как Docker volume).

Миграции накатываются автоматически через `ALTER TABLE` при старте — ручного alembic нет.

### Ключевые таблицы

| Таблица | Назначение |
|---|---|
| `users` | Профили участников, XP, стрики, уровни |
| `challenges` | Челленджи (5 типов) |
| `challenge_participants` | Присоединившиеся к чужим челленджам |
| `reports` | Отчёты о тренировках |
| `report_challenges` | Связь отчёт ↔ несколько челленджей |
| `votes` | Голоса за/против отчётов |
| `personal_records` | Личные рекорды по дистанции |
| `event_templates` | Шаблоны мероприятий |
| `events` | Конкретные мероприятия |
| `event_participants` | Регистрация на мероприятия |
| `weekly_tournaments` | Турниры |
| `tournament_participants` | Участники турниров с очками |
| `moderators` | Модераторы бота |

Поле `vote_message_id` в `reports` хранит ID vote-сообщения в чате — используется для снятия кнопок после апрува/реджекта.

## Тестирование

Скрипт `test_bot.py` тестирует бизнес-логику без живого Telegram (in-memory SQLite, мок бота):

```bash
# Все тесты
python test_bot.py

# Только турниры
python test_bot.py tournaments

# Только дайджест
python test_bot.py digest

# Дайджест + реальная отправка в группу (нужен .env)
python test_bot.py digest --send
```

В Docker:
```bash
docker compose exec bot python test_bot.py
```
